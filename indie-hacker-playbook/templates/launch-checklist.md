# Launch Checklist

## 4 Weeks Before Launch

- [ ] Landing page live with email capture
- [ ] Core product functional (MVP — not perfect)
- [ ] Pricing decided and displayed
- [ ] Payment integration working (test with real card)
- [ ] Analytics installed (Plausible, PostHog, or similar)
- [ ] Error tracking set up (Sentry or similar)

## 2 Weeks Before

- [ ] Product Hunt Coming Soon page created (if launching on PH)
- [ ] 5-10 beta users testing and giving feedback
- [ ] Screenshots and demo GIF/video created
- [ ] Landing page copy finalized
- [ ] Social media posts drafted (launch day + follow-ups)
- [ ] Email to existing list drafted
- [ ] Show HN post text written and reviewed
- [ ] Reddit communities identified (read their rules)
- [ ] Indie Hackers post drafted

## 1 Week Before

- [ ] Notify beta users about launch date — ask for support
- [ ] Share Coming Soon link with personal network
- [ ] Final bug fixes from beta feedback
- [ ] Prepare launch-day response templates (thank-you, FAQ answers)
- [ ] Queue up social media posts
- [ ] Brief any supporters on what to do launch day
- [ ] Test payment flow end-to-end one more time
- [ ] Test landing page on mobile

## Launch Day

- [ ] Product Hunt post goes live (if applicable)
- [ ] Show HN posted with maker description
- [ ] Tweet/X post with product link and story
- [ ] Indie Hackers "Show IH" post
- [ ] Reddit post in relevant subreddit(s)
- [ ] Email blast to list
- [ ] Personal messages to supporters asking them to check it out
- [ ] Monitor and respond to EVERY comment on every platform
- [ ] Fix any critical bugs reported immediately
- [ ] Share progress updates throughout the day

## 1 Week After Launch

- [ ] Thank-you post on all channels with results
- [ ] Follow up with every signup via email
- [ ] Compile feedback into a prioritized list
- [ ] Fix top 3 reported issues
- [ ] Write a "launch retrospective" post (great for content)
- [ ] Plan next launch (different channel or major update)

## Ongoing (Post-Launch)

- [ ] Publish a blog post or changelog for every update
- [ ] Share milestones publicly (first customer, $100 MRR, etc.)
- [ ] Engage daily in 1-2 communities where your users hang out
- [ ] Send monthly update email to users
- [ ] Plan next "launch moment" every 4-6 weeks
