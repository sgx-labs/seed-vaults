# Setting Up a Documentation Site with Docusaurus

## Prerequisites

- Node.js 18+ (`node --version`)
- npm or yarn
- Existing Markdown docs (or willingness to write some)

## Step 1: Create the Project (5 minutes)

```bash
npx create-docusaurus@latest my-docs classic
cd my-docs
npm start
```

Open http://localhost:3000. You should see the default Docusaurus site.

## Step 2: Replace Default Content (15 minutes)

Delete the default docs and add your own:

```bash
rm -rf docs/*
rm -rf blog/*
```

Copy your Markdown files into `docs/`:

```
docs/
  intro.md
  getting-started.md
  guides/
    deploying.md
    configuration.md
  api/
    authentication.md
    endpoints.md
```

## Step 3: Configure the Sidebar (10 minutes)

Edit `sidebars.js`:

```javascript
module.exports = {
  docs: [
    'intro',
    'getting-started',
    {
      type: 'category',
      label: 'Guides',
      items: ['guides/deploying', 'guides/configuration'],
    },
    {
      type: 'category',
      label: 'API',
      items: ['api/authentication', 'api/endpoints'],
    },
  ],
};
```

Or use auto-generated sidebars (simpler for large doc sets):

```javascript
module.exports = {
  docs: [{type: 'autogenerated', dirName: '.'}],
};
```

## Step 4: Update Site Config (10 minutes)

Edit `docusaurus.config.js`:

```javascript
module.exports = {
  title: 'My Project',
  tagline: 'Documentation for My Project',
  url: 'https://docs.myproject.com',
  baseUrl: '/',
  // ... rest of config
};
```

Key settings to update:
- `title` and `tagline`
- `url` (your production URL)
- `organizationName` and `projectName` (for GitHub Pages)
- `navbar` links
- `footer` links

## Step 5: Add Search (10 minutes)

For local search (no external service):

```bash
npm install @easyops-cn/docusaurus-search-local
```

Add to `docusaurus.config.js`:

```javascript
themes: [
  ['@easyops-cn/docusaurus-search-local', {
    hashed: true,
  }],
],
```

## Step 6: Deploy (15 minutes)

### GitHub Pages

```bash
npm run build
GIT_USER=<your-github-username> npm run deploy
```

### Vercel or Netlify

Connect your repo. Both auto-detect Docusaurus and configure build settings.

### Any Static Host

```bash
npm run build
# Upload the build/ directory to your host
```

## Step 7: Add CI (10 minutes)

```yaml
name: Deploy Docs
on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npm ci
      - run: npm run build
      - uses: peaceiris/actions-gh-pages@v4
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./build
```

## Total Time

About 75 minutes from zero to a deployed documentation site.

## Common Gotchas

- **Markdown frontmatter.** Docusaurus uses `sidebar_position` to order pages. Add it if auto-ordering isn't right.
- **File naming.** Use kebab-case for file names. Docusaurus generates URLs from file names.
- **Images.** Put images in `static/img/` and reference as `/img/filename.png`.
- **MDX.** Docusaurus supports MDX (Markdown + JSX). You can use React components in docs, but you don't have to.
