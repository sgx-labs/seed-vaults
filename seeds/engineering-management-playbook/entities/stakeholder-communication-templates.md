# Stakeholder Communication Templates

## Weekly Status Update

```
Subject: [Team Name] Weekly Update — Week of [Date]

**Summary**: [One sentence: what's the headline?]

**Key Deliverables**:
- [Deliverable 1]: [On track / At risk / Blocked] — [One sentence status]
- [Deliverable 2]: [On track / At risk / Blocked] — [One sentence status]
- [Deliverable 3]: [On track / At risk / Blocked] — [One sentence status]

**Wins This Week**:
- [Concrete achievement with impact]

**Risks & Blockers**:
- [Risk/blocker with owner and mitigation plan]

**Needs from Leadership**:
- [Specific ask, or "None this week"]
```

## Project Risk Escalation

```
Subject: [RISK] [Project Name] — [Risk Summary]

**Bottom Line**: [What's happening and what you need, in two sentences]

**Impact**: [Who is affected, quantify if possible]
**Likelihood**: [High / Medium / Low]
**Timeline**: [When will this become a problem?]

**Options**:
1. [Option A]: [Description, tradeoff, timeline]
2. [Option B]: [Description, tradeoff, timeline]
3. [Do nothing]: [What happens if we don't act]

**My Recommendation**: Option [X] because [reason].

**Decision Needed By**: [Date]
```

## Bad News Delivery

```
Subject: [Project/Issue] — Status Change

**What happened**: [Direct, factual description. No hedging.]
**Impact**: [Who/what is affected and how]
**Root cause**: [What caused it, or "under investigation"]
**What we're doing**: [Immediate actions, with owners and timelines]
**What we need**: [Help, decisions, resources, or nothing]
**Next update**: [When they'll hear from you next]
```

## Headcount Request

```
Subject: Headcount Request — [Role Title]

**What**: [Number] [Role] for [Team]
**Why**: [Business justification in 2-3 sentences. Focus on what we CAN'T do without this hire.]

**Impact of NOT hiring**:
- [Specific consequence 1]
- [Specific consequence 2]

**Impact of hiring**:
- [Specific outcome 1]
- [Specific outcome 2]

**Budget**: [Estimated comp range]
**Timeline**: Need by [date], expect [X weeks] to fill

**Level**: [IC2-IC4, M1-M2, etc.]
```

## Quarterly Team Review

```
Subject: [Team Name] — Q[X] Review

**Highlights**:
- [Top 3 achievements with measurable impact]

**Metrics**:
- Deployment frequency: [X] (was [Y])
- Incident count: [X] (was [Y])
- Team satisfaction: [X]/5 (was [Y]/5)

**Challenges**:
- [Key challenge and what we're doing about it]

**Next Quarter Focus**:
- [Top 2-3 priorities]

**Ask**:
- [What you need from leadership for next quarter]
```

## Tips for All Templates

- **BLUF (Bottom Line Up Front)**: Lead with the conclusion, not the context
- **Bold the key information**: Busy stakeholders scan, they don't read
- **One page maximum**: If you need more, attach a doc and link to it
- **Include a specific ask**: Don't make the reader figure out what you want
